# fire-safety-personnel-tutor

消防設備師／士國家考試備考 Claude Code plugin：

- **exam-tutor** — 家教模式：以消防安全設備為主體、依 `reference/設備條文索引.md` 課綱地圖連續出題（先問後等再解）、收作答、0–100 分批改，講解時引用法條全文（條／項／款／目階層）；含快速抽考（3~5 題）與弱點複習模式。
- **exam-trend-forecast** — 考前猜題（**只負責猜題**）：結合近 10 年加權考點分布與近 12–24 個月的法規修正、行政函令、重大時事，產出「法規脈動／技術實務／時事預測」之**猜題範圍**——每項含具體範圍（法規＋條文／設備）、依據推理與信心程度。例：過去一年發生瓦斯氣爆重大案件 → 今年可能考瓦斯漏氣火警自動警報設備之設置規定與檢知器裝設；發生施工中建築物火災 → 可能考消防法之施工中消防防護計畫。後續的出題、清單、懶人包由其他功能接手（報告尾指路）。
- **statute-memorizer** — 法規記憶助手：整理易混淆數字／時限／罰則成記憶卡與口訣、產生跨法規對照表（內建 8 張高頻對照表＋即時生成），並依猜題範圍濃縮**考前必背懶人包**。
- **study-planner** — 讀書計畫與猜題清單：以消防安全設備為主體、按師/士命題頻率主動規劃研讀順序（設置標準 → 公危管理 → 消防法 → 檢修基準 → 其他，依實際頻率排序），產出可勾銷清單或逐週排程（考前自動預留總複習與猜題），並與出題進度銜接；另管理**可勾銷猜題清單**（跨對話接續）。
- **exam-archive** — 檔案庫查詢：列出題庫清單、提供指定年度科目的原卷 PDF、標準答案卷或單題原文。

### Slash commands

| 指令 | 功能 |
|------|------|
| `/抽考 [範圍]` | 快速抽考一輪 3~5 題（零碎時間用） |
| `/弱點複習 [範圍]` | 依作答紀錄優先重考常錯考點 |
| `/記憶重點 <主題>` | 整理記憶卡／口訣／易混淆組 |
| `/對照表 [主題]` | 跨法規對照表（留空列出內建清單） |
| `/猜題 [範圍]` | 產出猜題範圍（考點＋依據＋信心）——exam-trend-forecast |
| `/猜題清單` | 猜題範圍存成可勾銷清單、跨對話接續——study-planner |
| `/懶人包 [範圍]` | 考前必背懶人包——statute-memorizer |
| `/申論猜題 [科目]` | 申論題猜題＋答題架構＋擬答——exam-tutor |
| `/讀書計畫 [日期 時數]` | 依命題頻率排研讀順序；附考試日期＋每週時數則產逐週排程 |
| `/設定` | 設定應考等別（師/士）、弱點記錄模式 |

所有內容與產出均為繁體中文。

## 使用範例

### 出題家教（exam-tutor）

> **你**：考我滅火器
> **Claude**：依 `reference/設備條文索引.md` 從滅火器第一條相關條文開始連續出題——§14 應設場所 → §31 設置規定 → 公危 §223／§224／§225／§228 → 檢修基準第 1 章 → 延伸知識（四種乾粉化學反應式，標注「非法規條文」）。出題後**停下等你作答**；作答後給 0–100 分批改，正解每一論點附法源出處並呈現條文全文（條／項／款／目階層＋版本日期）。

沒指定設備時，Claude 會主動提議（依讀書計畫 → 弱點 → 頻率），或列出設備清單讓你挑。

### 快速抽考（/抽考）

> **你**：`/抽考 警報系統`
> **Claude**：一輪 3~5 題快問快答，一題一答一講（給分＋正解＋關鍵條文出處），輪末報成績並問要不要再一輪。適合通勤零碎時間。

### 弱點複習（/弱點複習）

> **你**：`/弱點複習`
> **Claude**：讀本機 `progress.json` 的 `weak_tally`，列出你最常錯的考點（例：滅火器 ×3、設置標準§31 ×2），優先重考曾答錯的題目，答對調降權重、答錯稍後再出變形題確認。

### 系統複習（exam-tutor 複習模式）

> **你**：帶我看水系統的規定
> **Claude**：不逐題抽問，改整批講解水系統脈絡：設置標準對應條文＋各設備＋檢修要領＋常考數字公式，段落後附簡短自我檢測。

### 記憶重點（/記憶重點）

> **你**：`/記憶重點 檢修申報期限`
> **Claude**：引內建對照表 01 做成記憶卡與口訣（例：「甲類半年一檢；一~三目＝3、9 月底」），易混淆組並列點破（每半年檢修 vs 每半年訓練）。可存入 `notes/` 累積筆記庫。

### 跨法規對照表（/對照表）

> **你**：`/對照表`
> **Claude**：列出內建 8 張清單（檢修申報、場所門檻、水系統數值、警報距離、化學藥劑、避難收容、消防法罰則、公危管制量）。
> **你**：`/對照表 各設備緊急電源分鐘數`
> **Claude**：不在內建範圍 → 即時從 `statutes/` 逐條整理生成，標注「即時生成，請核對現行版」。

### 考前猜題（/猜題）——只給猜題範圍

> **你**：`/猜題`
> **Claude**：以近 10 年指數衰減加權頻率建歷史底，搜尋近 12–24 個月修法（含**修法理由**）、函令、重大時事，產出「法規脈動／技術實務／時事預測」之**猜題範圍**，每項含範圍、依據與信心程度。例：
> > 【時事預測｜信心：中】過去一年發生○○瓦斯氣爆案 → 建議優先準備**瓦斯漏氣火警自動警報設備**設置規定與檢知器裝設（設置標準 §21、§140～§145-1）
> > 【法規脈動｜信心：高】消防法 §13 修正施工中消防防護計畫 → 建議優先準備**施工中消防防護計畫**訂定與備查程序
> 報告尾指路：想練題 → `/申論猜題` 或「依猜題範圍考我」；想追蹤 → `/猜題清單`；想背重點 → `/懶人包`。猜題功能本身不代做這些。

### 猜題清單（/猜題清單）——study-planner 負責

> **你**：`/猜題清單`
> **Claude**：把最近一次猜題範圍存成 checkbox 清單檔 `forecasts/猜題清單_師115.md`；下次開新對話說「把第 2 項勾掉」可接續勾銷，或重跑 `/猜題` 後增補新動態。

### 申論猜題＋擬答（/申論猜題）——exam-tutor 負責

> **你**：`/申論猜題 消防法規`
> **Claude**：依猜題範圍（或你指定的科目）預測申論題目＋答題架構（破題 → 法源 → 論述 → 結論）＋擬答範例。**擬答每一論點附法源依據**，例：
> > 依《消防法》第 9 條第 1 項第 1 款，高層建築物應委託檢修專業機構辦理定期檢修…（113-11-29 修正版）
> > ※ 非法源：實務見解——申報實務上多於期限前一個月完成檢修以保留改善時間。

### 考前必背懶人包（/懶人包）——statute-memorizer 負責

> **你**：`/懶人包`
> **Claude**：依猜題範圍（無則取該等別加權排名前 20 考點）濃縮成一份文件：考點 → 必背條文重點（數字、關鍵句）→ 修法新點 → 一句記憶術，存 `forecasts/考前必背懶人包_師115.md`。

### 讀書計畫（/讀書計畫）

> **你**：`/讀書計畫`
> **Claude**：依你的等別取命題頻率，由高至低編排單元（設置標準（按設備切單元，火警自動警報最先）→ 公危管理 → 消防法 → 檢修基準 → 其他），每單元附條文範圍與建議行動（「考我〈設備〉」／`/記憶重點`／對照表連結），產出可勾銷清單。
> **你**：`/讀書計畫 考試 115-06-14 每週 10 小時`
> **Claude**：改產逐週排程——高頻單元配較多時數，**考前最後 1~2 週自動預留弱點總複習與猜題**；之後出題時 exam-tutor 會優先提議計畫中下一個未完成單元，落後時可要求重排。

### 設定（/設定）

> **你**：`/設定`
> **Claude**：顯示現值並可修改：應考等別（師＝六科全申論／士＝四科申論＋測驗混合）、弱點記錄模式（自動記錄本機／產出筆記自存／不記錄）、資料目錄（預設 `~/.fire-safety-tutor/`）。各功能初次使用會自動問一次並記住，之後不再重複詢問。

### 檔案庫查詢（exam-archive）

> **你**：給我 113 年設備師消防法規的原卷
> **Claude**：從 `corpus/index.json` 定位並提供 `pdf/師/113/0802_消防法規.pdf` 與 md 全文；含測驗題科目優先給更正後答案卷。也可依標籤查題：「列出考過緩降機的題目」。


## 題庫現況

**設備師（師）**：民國 100–115 年六科、**設備士（士）**：100–115 年四科，共 160 卷，原卷 PDF 與 md 均全數入庫。md 與 PDF 分置於 `corpus/md/` 與 `corpus/pdf/`。全庫共 3680 題，已逐題標籤（題型／系統／設備／法規／**條號**／知識領域／旗標）；各題 inline `🏷️` 標籤為唯一真相來源，索引由 `scripts/rebuild_index.py` 反建（`corpus/tags_index.json`）、`scripts/analyze_corpus.py` 產生統計（`corpus/tags_summary.json`、`corpus/命題頻率分析.md`），維度說明見 `docs/設計_題目標籤系統.md`，各腳本說明見 [`scripts/README.md`](scripts/README.md)。

**法條**：命題大綱所列法規（`statutes/index.md` 第一～四節）已全數整理為現行全文 md。

## 資料準備

1. **考古題**：新增年度或士類試卷時，依 [`corpus/INGEST.md`](corpus/INGEST.md) 自考選部下載官方 PDF、轉成 md、更新 `corpus/index.json`。
2. **法條**：依 [`statutes/index.md`](statutes/index.md) 的清單與格式規範，自全國法規資料庫／消防署整理現行全文 md 放入 `statutes/`（命題大綱所列法規已全數入庫並完成人工核對；各檔檔首附「📌 免責聲明」，引用前請依官方公告核對）。

skill 對 `corpus/`、`statutes/` 與 `reference/` 一律唯讀。使用者設定（應考等別、弱點記錄模式）與學習進度存放於使用者自己的電腦（預設 `~/.fire-safety-tutor/`，plugin 之外），初次使用時詢問一次並記住；規格見 [`reference/user-config-spec.md`](reference/user-config-spec.md)。

## 設計原則

- **以 corpus 取題型風格，以 statutes／即時搜尋取法源**：考古題只反映當年法規快照（`law_snapshot`），不可當現行法引用。
- **先問、後等、再解**：使用者作答前不給答案、不更新進度。
- **誠實性**：出題與批改前先自我驗證；趨勢預測附理由與信心程度，不呈現假確定。
- **優雅退場**：題庫未放入、法條缺漏、無網路等情況均降級運作並說明，缺條文且抓不到官方全文時停下求助。
