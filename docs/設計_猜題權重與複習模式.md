# 設計筆記：猜題的法條出題權重　與　複習模式

> 記錄使用者需求（2026-06）。**前提**：待 `statutes/` 法條 md 上傳齊全後，下列方法即可full運作；未齊全前 skill 退化為主題／設備層級的粗估（見各節「退化」）。

## 一、需求摘要（使用者原話重點）

1. 歷屆試題中各法條權重不同：有的考很多次，有的只考一次。
2. **猜題（exam-trend-forecast）**：應依此權重預測，避免「少考的法條反被預測為高機率」或反之。
3. 權重方法傾向**加權平均法**：**近幾年出得多的，被出題機率高一些**（近年加重）。
4. **複習（exam-tutor 複習模式）**：仍**按不同系統**，一次複習整個消防設備系統的相關規定（系統性，而非逐題抽問）。

## 二、猜題：法條出題權重（近年加權頻率）

### 統計窗口
近 **10 年**（涵蓋命題週期）。注意與 exam-tutor 出題頻率排序的**五年**窗口目的不同：出題求「近期熱區」，猜題求「週期＋趨勢」，故窗口較長。

### 計算步驟
1. **建立 (法規 → 各年出題次數)**：掃描 `corpus/` 每份試卷每一題，判定其所考法規。
   - 消防法規測驗題多於題幹明示「依○○法第 X 條」「依○○設置標準」→ 可直接對應到**法規**，能辨識條號者再細到**條**。
   - 申論題依題幹關鍵字與 `statutes/index.md` 法規名稱對應。
2. **年份權重（指數衰減）**：`w(y) = r^(Y_now − y)`，建議 `r = 0.8`（可調；r 越小越偏重近年）。
3. **法規加權分數**：`S(g) = Σ_y n(g,y)·w(y)`，n(g,y) 為法規 g 於 y 年之出題次數。
4. **正規化為基本盤機率**：`P(g) = S(g) / ΣS`，作為「歷史基本盤」排序。
5. **與動態三維度合成**：基本盤 P(g) 提供常態機率；今年若有**修法／函令／時事**觸及某法規，對其機率上調並於理由說明（動態加成）。長期低頻法規不因單一年雜訊被高估。
6. **輸出**：每個法規／主題附「加權頻率排名、近 N 年出現分布、信心程度」。

### 為何用加權平均（而非純次數或純最近一年）
- 純總次數：忽略趨勢，舊年代熱門但已少考者被高估。
- 純最近一年：雜訊大，偶發題被當常態。
- 加權平均：兩者折衷，符合使用者「近年加重、但仍看整體分布」之意。

### 退化（資料未齊）
`statutes/` 未齊或無法對應到法規層級時 → 退化為**主題／設備層級**加權頻率（用題幹關鍵字 ＋ `corpus/equipment_index.md`），並標注「法條對應未完整，權重為粗估」。

### 護欄
權重為**輔助排序非保證**；仍附信心程度與資料截至日期，不得呈現「一定考」之假確定（沿用 exam-trend-forecast 誠實性護欄）。

## 三、複習：按系統整批複習（exam-tutor 複習模式）

當使用者要「**複習／整理**」而非「被考」時，**不採**單一設備連續抽問，改：

- **按系統**整批複習：水系統 / 化學系統 / 警報系統 / 避難系統，及共通的**消防法規總論**。
- 一次帶完整個系統的相關規定：設置標準對應條文 ＋ 該系統各設備 ＋ 檢修要領 ＋ 常考數字／公式，形成系統性脈絡。
- 可結合本文件第二節權重：系統**內部**先複習高權重（常考）條文／設備，但仍**覆蓋整個系統**避免遺漏。
- 複習段落後可附「自我檢測」小測，但複習模式以**講解整理為主、測驗為輔**（與出題模式相反）。
