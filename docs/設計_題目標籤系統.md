# 設計：題目標籤系統（per-question tagging）

## 目的（使用者需求）

> 「當題目都上傳後，幫每題標上標籤，方便之後快速依照不同系統和消防安全設備定位那些題目，而不用每次掃過所有題目。或是題型例如申論題跟選擇題，或其他任何應用。」

讓 skill（猜題 exam-trend-forecast、家教 exam-tutor）與使用者能**不必掃過全部題目**，即可依「系統／設備／題型／…」快速定位相關題目，作為：

- 家教「按系統一次複習整個系統」時，秒抓該系統歷年所有題目。
- 猜題統計「設備出題頻率」時，直接以標籤計數，免去逐題語意判讀。
- 使用者自訂查詢（如「只看計算題」「只看更正答案題」「只看建築技術規則相關題」）。

## 執行時機

**待全部考古題（師 100–115、士 100–115）md 轉檔完成後，一次性套用。** 截至本文件，士 100–105 尚未轉檔，故先定義架構，待補齊後執行（見「待辦」）。

## 標籤維度（taxonomy）

每題可同時擁有多個維度的標籤。維度與允許值如下。

### 1. 題型 type（必填，單值）

- `申論題`
- `測驗題`

### 2. 系統 system（必填，可多值）

沿用 `corpus/equipment_index.md` 之四大系統：

- `滅火設備`
- `警報設備`
- `避難逃生設備`
- `消防搶救上之必要設備`

跨系統題（如綜合題、法規通則）可標多個或標 `綜合/通則`。

### 3. 消防安全設備種類 equipment（可多值）

細到單一設備，值域取自 `equipment_index.md`（滅火器、室內消防栓、自動撒水設備、火警自動警報設備、緊急廣播設備、瓦斯漏氣火警自動警報設備、標示設備、避難器具、緊急照明設備、連結送水管、消防專用蓄水池、排煙設備、緊急電源插座、無線電通信輔助設備、防災監控系統綜合操作裝置、119 火災通報設備…）。非設備類題目（純法規程序、火災學理）此欄留空。

### 4. 法規依據 law（可多值）

題幹明確指向之法源，便於「修法影響」追蹤。常見值：

- `各類場所消防安全設備設置標準`
- `消防法`／`消防法施行細則`
- `各類場所消防安全設備檢修及申報作業基準`（含「檢修基準」）
- `消防安全設備測試報告書測試方法及判定要領`
- `公共危險物品及可燃性高壓氣體製造儲存處理場所設置標準暨安全管理辦法`
- `建築技術規則`
- `消防設備師及消防設備士管理辦法`／`消防安全設備檢修專業機構管理辦法`
- `消防幫浦加壓送水裝置等及配管摩擦損失計算基準`
- `防火牆及防火水幕設置基準`／`可燃性高壓氣體儲存場所防爆牆（防護牆）設置基準`
- 各設備之 `認可基準`
- `NFPA`（國外標準引用）

### 4b. 法條（條號）article（可多值，自動擷取）

在「法規依據」之外再細到**條號**，便於依單一條文定位歷年考題（如「設置標準第 12 條」出過幾題）。

- 由 `build_tags.py` 以正規表示式擷取題幹中**明確引用**之「法名＋第 X 條（之 Y）」，以最近出現之法規短名作前綴。
- 僅在可歸屬法源時納入（題幹未提及法名之純概念題不標，避免雜訊）。
- 短名：`消防法`、`施行細則`、`設置標準`、`公危辦法`、`建築法`、`建築技術規則`、`檢修基準`、`測試報告書`、`設備人員管理辦法`、`檢修機構管理辦法`、`幫浦計算基準`、`認可基準`。
- 格式範例：`設置標準第12條`、`消防法第15條之5`、`建築法第34條之1`。中央索引維度：`by_article`。

### 5. 知識領域 topic（火災學等學理科適用，可多值）

火災學概要與部分學理題用，值如：`燃燒理論`、`火災化學/化學計量`、`爆炸`、`粉塵爆炸`、`煙控/煙流`、`熱傳`、`滅火原理`、`電氣火災`、`靜電`、`危險物品分類`、`火災調查`、`特殊場所火災`。

### 6. 其他輔助標籤 flags（可多值，自動可判定）

- `計算題`：含數值運算、公式套用者。
- `找錯誤`／`找正確`：題幹為「何者錯誤」vs「何者正確/最適當」。
- `圖形題`：選項或題幹為圖（需對照 PDF），對應 `has_figure`。
- `更正答案`：該題有更正（給分調整、一律給分、複選給分）。
- `跨科`：考點橫跨多科（如法規×設備）。

## 儲存方式（雙軌）

1. **題目內嵌（inline）** — 在每題標準答案行下方加一行引用區塊，人可讀、grep 可搜：

   ```
   > 🏷️ 測驗題｜滅火設備｜自動撒水設備｜各類場所消防安全設備設置標準｜計算題
   ```

   申論題同理標於題目末。維度間以全形直線 `｜` 分隔；多值同維度以頓號 `、` 分隔。

2. **中央索引（machine-readable）** — 由建置腳本掃描所有 md，彙整成 `corpus/tags_index.json`，結構示意：

   ```json
   {
     "by_equipment": { "自動撒水設備": ["士/106/0910#Q18", "師/115/..#Q.."] },
     "by_system":    { "滅火設備": [ ... ] },
     "by_type":      { "申論題": [ ... ], "測驗題": [ ... ] },
     "by_law":       { "消防法": [ ... ] },
     "by_article":   { "設置標準第12條": [ ... ], "消防法第13條": [ ... ] },
     "by_flag":      { "計算題": [ ... ], "更正答案": [ ... ] }
   }
   ```

   題目參照格式：`<等別>/<年>/<科目代碼>#Q<題號>`（申論用 `#甲<n>`）。skill 讀此檔即可 O(1) 定位，免掃全庫。

> 內嵌為單一事實來源（source of truth），中央索引由腳本再生，避免兩處不一致。
