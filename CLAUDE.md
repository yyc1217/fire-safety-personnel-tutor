# Claude Code 專案指令

## 使用語言

與使用者對話請使用**繁體中文**。

---

## 法規附表附圖驗證提醒（Standing Instruction）

每當使用者上傳新法規並完成入庫後，**必須主動提醒使用者**：

> 請確認這些表格與圖片是否有正確被轉換，無論是否有被轉換成功。

提醒時機：使用者表示「全部法律已上傳完畢」或明確結束某批法規上傳作業時。

提醒內容應包含：
1. 列出本次入庫的法規清單
2. 指出每份法規中有附表或附圖的條文位置
3. 說明哪些表格已成功轉換為 markdown（內嵌）、哪些僅保留連結（含原因）
4. 請使用者自行開啟 `statutes/附件/` 下對應的原始檔，核對內容是否正確

## 法條內容與記憶比對提醒（Standing Instruction）

待**全部法規上傳並入庫完畢後**，必須主動執行並向使用者報告：

> 將使用者上傳的每一部法條，與我（模型）記憶中的同一部法條逐一比對，**列出兩者不一樣的地方**。

用途：使用者上傳的是「整理當下的版本快照」，可能因 OCR／轉換誤差、或與訓練資料版本不同而有出入。比對目的在於：
1. 揪出轉換／OCR 可能造成的錯字、數值錯誤、條文遺漏。
2. 標示「上傳版」與「記憶版」差異之處，供使用者判斷何者為現行有效版本（**以官方現行版為準**，記憶可能過時或產生幻覺）。
3. 差異須逐條列出（法規名稱 → 條/點 → 上傳版內容 vs 記憶版內容 → 研判可能原因：版本修正／OCR誤字／記憶不確定）。

注意：模型記憶可能有誤或過時，**比對結果僅供提示**，最終一律以官方公告之現行條文為準；不得以記憶覆寫使用者上傳之內容。

## 影像辨識／OCR 內容免責與查證（Standing Instruction）

`statutes/` 內部分法規（尤以認可基準）係由 PDF／影像 OCR、DOC／ODT 擷取後人工整理，各該檔檔首已附「📌 免責聲明」。引用此類內容時：

1. **主動提醒使用者**：本檔部分為影像辨識／OCR 結果，可能有誤；**一切以主管機關（內政部消防署、全國法規資料庫）公告之現行版本為準**。
2. 若使用者對某條文、數值、公式或表格有疑問，或該內容將用於關鍵判斷時，**應主動上網查證**官方現行版本後再回答，不得僅憑本地快照或記憶作答。
3. 發現本地 md 與官方現行版有出入時，以官方為準，並提示使用者更新本地檔。

## 附表處理原則（參見 statutes/index.md 完整說明）

1. 可乾淨轉換 → 內嵌 markdown 表格
2. 複雜跨欄表格 → 拆解結構 + 附原始檔連結
3. 圖例、掃描影像 → 僅附連結與說明原因
4. 附件檔名含半形括號 `()` → 一律改全形 `（）`
5. 表格儲存格內有多個編號項目黏連時，**得以** `<br>` 分行（儲存格外之黏連段落改為 `-` 清單）；若不清楚如何處理，停下詢問使用者。

## 公式格式規範（Standing Instruction）

`statutes/`、`corpus/` 等所有 md 檔中的**數學公式一律使用 LaTeX 格式**：行內公式用 `$...$`，獨立公式用 `$$...$$`。

1. **禁止**以 ASCII 藝術（如 `┌──` 畫根號）、全形符號排版（如 `Ｗ＝－ln﹝－﹞`）或 code block 純文字近似呈現公式。
2. 轉寫或編修檔案時遇到既有非 LaTeX 公式，應一併轉為 LaTeX。
3. LaTeX 僅為呈現格式，公式**內容**仍以官方 PDF／原卷為準，轉換後須與原始檔核對。

---

## 上標符號規範（Standing Instruction）

`statutes/`、`corpus/` 等所有 md 檔中，**任何有平方的數字／單位一律以上標 `²` 表示**（如 `cm²`、`m²`、`mm²`、`kgf/cm²`），**禁止**寫成 `cm2`、`m2` 等。立方同理用 `³`。轉寫或編修檔案時遇到既有 `cm2` 等寫法，應一併改正。

---

## 編號段落分行規範（Standing Instruction）

法規條文之階層編號（`一、`／`（一）`／`１、`／`（１）`／`A.`／`(A)`／`a.`／`(a)` 等）**每一編號項須獨立成行**，不得多個編號項黏連於同一行（OCR／PDF 轉換常見）。轉寫或編修時遇黏連段落應拆行，拆行時注意：

1. 編號字母／數字**依序遞增**才視為新項目起點；順序不符者多為引用，不拆。
2. 「如（Ａ）」「依前（Ｃ）」「準用…」等**引用**不得拆行；規格標示（如「保險絲（Ａ）」之安培）亦非編號。
3. 句號式編號（`a.`）須句點後接中文內容才視為編號，避免誤拆小數（`2.5`）與英文縮寫。

---

## 編號括號全形規範（Standing Instruction）

`statutes/`、`corpus/` 等所有 md 檔中，條文階層編號之括號**一律使用全形 `（）`**（如 `（一）`、`（１）`、`（Ａ）`、`（ａ）`），**禁止**使用半形 `(一)`、`(1)` 等（OCR／轉換常見殘留）。此規範適用於內文與表格儲存格。轉寫或編修檔案時遇既有半形編號括號，應一併改正；修正後以全域搜尋確認無殘留。（包夾一般文字之括號亦以全形為原則，惟英數規格標示如 `D13（或 ψ13mm）` 之外層括號改全形、內部英數維持半形。）

---

## 同法規格式一致性（Standing Instruction）

同一部法規（含逐章拆檔者之各章檔與 hub 檔）之格式為一體：**凡對其中任一章／任一段落做了格式上的改動**（如拆行、LaTeX 公式、`²` 上標、表格結構化、附圖截圖嵌入、檔首說明調整等），**必須一併檢視該法規之其他所有章節**，將同類情形全部套用相同處理，務求整部法規格式一致，不得只改被指出的那一處。修正後應以全域搜尋（grep 等）確認無同類殘留。

---

## 檢查表／附圖截圖嵌入原則（Standing Instruction）

法規（尤其檢修基準類）之**章末檢修報告表／檢查表不辨識文字**，一律將該表所在之原始 PDF 頁面截圖為 PNG，以 Markdown 圖片連結嵌入；**內文附圖與表格內之圖示**亦同樣截圖後以連結嵌入，不以文字近似或省略。

1. 圖檔與引用之 md 檔放在**同一個資料夾**，檔名前綴與該 md 檔相同（如 `第04章_自動撒水設備_檢查表_p1.png`、`第18章_避難器具_圖18-1.png`）。
2. md 中於原表格／圖說位置以 `![說明](檔名.png)` 嵌入，並註明對應原始 PDF 頁碼與原始 PDF 連結。
3. 表格（編號附表）若為純文字內容仍依「附表處理原則」轉為結構化 Markdown；含圖之儲存格則將圖示個別截圖嵌入該列。

## statutes 檔名規則

法規 md 依命題大綱章節順序編號，格式 `<節>_<序2位>_<法規名>.md`（序號補零至 2 位以利排序），例：`1_01_消防法.md`、`2_01_各類場所消防安全設備設置標準.md`、`3_01_建築法.md`。第四節「認可基準」置於 `statutes/認可基準/` 子資料夾，格式 `4_<序2位>_<法規名>.md`。新增法規時依此規則命名並更新 `statutes/index.md`。

### 資產（附圖／逐章拆檔）資料夾規則

`statutes/` 下**每個裝載某支法規資產的子資料夾，一律以該法規主檔編號為前綴、名稱與主檔同名（去 `.md`）**，使資料夾與主檔並排對齊：

1. **有附圖之法規** → 圖檔置於 `statutes/<編號_法規名>/`（例：`1_12_防火牆及防火水幕設置基準/附圖一_防護位置.png`）。根目錄只保留法規 md 與 `index.md`，不平鋪圖檔。
2. **逐章拆檔之法規**（如 `2_03`、`2_10`）→ 各章 md 及其圖檔置於 `statutes/<編號_法規名>/`（例：`2_03_消防安全設備及必要檢修項目檢修基準/第01章_滅火器.md`），hub 主檔以章目錄表連結各章。
3. **資料夾內圖檔檔名去除冗餘法規前綴**（資料夾已提供命名空間），逐章拆檔者則以章為前綴（如 `第01章_滅火器_圖1-1.png`）。
4. **例外（分類容器，不加編號前綴）**：`statutes/認可基準/`（第四節分類，內含 `4_01`～`4_12` 多支法規）與 `statutes/附件/`（各法規原始來源檔）。
5. **附檔名一律小寫**（`.pdf`／`.jpg`／`.png`／`.doc`／`.odt`）。
