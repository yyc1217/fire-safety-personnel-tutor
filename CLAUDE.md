# Claude Code 專案指令

## 使用語言

與使用者對話請使用**繁體中文**。

---

## 法規附表附圖驗證提醒（Standing Instruction）

每當使用者上傳新法規並完成入庫後，**必須主動提醒使用者**：

> 請確認這些表格與圖片是否有正確被轉換，無論是否有被轉換成功。

提醒時機：使用者表示「全部法律已上傳完畢」或明確結束某批法規上傳作業時。

提醒內容應包含：
1. 列出本次入庫的法規清單
2. 指出每份法規中有附表或附圖的條文位置
3. 說明哪些表格已成功轉換為 markdown（內嵌）、哪些僅保留連結（含原因）
4. 請使用者自行開啟 `statutes/附件/` 下對應的原始檔，核對內容是否正確

## 法條內容與記憶比對提醒（Standing Instruction）

待**全部法規上傳並入庫完畢後**，必須主動執行並向使用者報告：

> 將使用者上傳的每一部法條，與我（模型）記憶中的同一部法條逐一比對，**列出兩者不一樣的地方**。

用途：使用者上傳的是「整理當下的版本快照」，可能因 OCR／轉換誤差、或與訓練資料版本不同而有出入。比對目的在於：
1. 揪出轉換／OCR 可能造成的錯字、數值錯誤、條文遺漏。
2. 標示「上傳版」與「記憶版」差異之處，供使用者判斷何者為現行有效版本（**以官方現行版為準**，記憶可能過時或產生幻覺）。
3. 差異須逐條列出（法規名稱 → 條/點 → 上傳版內容 vs 記憶版內容 → 研判可能原因：版本修正／OCR誤字／記憶不確定）。

注意：模型記憶可能有誤或過時，**比對結果僅供提示**，最終一律以官方公告之現行條文為準；不得以記憶覆寫使用者上傳之內容。

## 影像辨識／OCR 內容免責與查證（Standing Instruction）

`statutes/` 內部分法規（尤以認可基準）係由 PDF／影像 OCR、DOC／ODT 擷取後人工整理，各該檔檔首已附「📌 免責聲明」。引用此類內容時：

1. **主動提醒使用者**：本檔部分為影像辨識／OCR 結果，可能有誤；**一切以主管機關（內政部消防署、全國法規資料庫）公告之現行版本為準**。
2. 若使用者對某條文、數值、公式或表格有疑問，或該內容將用於關鍵判斷時，**應主動上網查證**官方現行版本後再回答，不得僅憑本地快照或記憶作答。
3. 發現本地 md 與官方現行版有出入時，以官方為準，並提示使用者更新本地檔。

## 附表處理原則（參見 statutes/index.md 完整說明）

1. 可乾淨轉換 → 內嵌 markdown 表格
2. 複雜跨欄表格 → 拆解結構 + 附原始檔連結
3. 圖例、掃描影像 → 僅附連結與說明原因
4. 附件檔名含半形括號 `()` → 一律改全形 `（）`

## statutes 檔名規則

法規 md 依命題大綱章節順序編號，格式 `<節>_<序2位>_<法規名>.md`（序號補零至 2 位以利排序），例：`1_01_消防法.md`、`2_01_各類場所消防安全設備設置標準.md`、`3_01_建築法.md`。第四節「認可基準」置於 `statutes/認可基準/` 子資料夾，格式 `4_<序2位>_<法規名>.md`。新增法規時依此規則命名並更新 `statutes/index.md`。
