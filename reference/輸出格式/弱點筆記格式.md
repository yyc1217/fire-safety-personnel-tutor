# 弱點筆記格式範本（exam-tutor 統一輸出，weakness_tracking = "notes"）

> 本檔定義 `notes` 模式下 exam-tutor 產出之弱點筆記格式，為此輸出的**格式唯一真相**。筆記**日後會由使用者提供回來供弱點複習選題**，格式須固定才能對回標籤索引。存 `notes/<YYYY-MM-DD>_<主題>弱點筆記.md`（命名見 `user-config-spec.md`「檔案命名慣例」）。

---

## 一、版面

```markdown
# 弱點筆記（{等別}）｜{YYYY-MM-DD}｜{主題：設備／系統／科目}

> 由 exam-tutor 產出。弱點複習時把本檔提供給 Claude 即可據以選題。

## 弱點清單

### {考點中文名}（錯 {n} 次）
<!-- tags: by_equipment:{設備} by_article:{法規條號} -->
- 題目參照：`{等別/年/科目代號#題號}`、…
- 錯誤型態：{一句描述錯在哪}
- 關鍵條文：{法規簡稱 §條號}（{該條關鍵句或公式}）

### {下一個考點}（錯 {n} 次）
…

## 本輪練習紀錄（選填，人讀用）

- {日期}｜{範圍}｜{得分摘要}
```

## 二、規則

1. 每個弱點考點一個 `###` 分節，**標題括號記累計錯誤次數**（對應 `weak_tally` 語意）。
2. **內部 tag 藏在 HTML 註解**：每個考點標題下緊接一行 `<!-- tags: ... -->`（key 對齊 `corpus/tags_index.json` 維度：`by_equipment:`／`by_article:`／`by_system:`）。**這些代碼是給 AI 讀回選題用的，對使用者沒有意義**——一律放註解（渲染時不可見），**不得以可見文字輸出**；讀回選題時以註解行為準，註解缺漏時改由考點中文名與「關鍵條文」之條號推回維度。
3. `題目參照：` 沿用 `q_id` 格式（`等別/年/科目代號#題號`），供優先重考曾答錯之原題。
4. 公式一律 LaTeX。同主題多次練習：使用者可自行併檔或累積多檔，讀回時以各檔 `###` 標題彙總次數。
