# 申論猜題擬答格式範本（exam-tutor 統一輸出）

> 本檔定義 `/申論猜題`（exam-tutor「申論猜題＋擬答模式」）的統一輸出格式，為此產出的**格式唯一真相**，每次逐區塊比照。內容面規範（法源硬性規則、逐步推導）依 exam-tutor SKILL.md 該模式段落；本檔只固定**版面**。

---

## 一、整體結構（每科一份）

```markdown
# 申論猜題擬答（{等別}{民國年}）｜{科目}

> 資料截至 {YYYY-MM-DD}。範圍來源：{使用者指定之考點｜本次猜題範圍（{趨勢報告檔名}）｜依{等別}加權頻率就地選取}。
> 擬答引用本地法規快照（版本日期見各引用處）；{關鍵條文已上網核對現行版｜未經線上即時核對}。

## 預測題一：{考點中文名}

…

## 預測題二：{考點中文名}

…
```

1. 標題含等別與民國年（＝目標考試年，算法依 `user-config-spec.md`），與存檔名 `申論猜題擬答_<等別><民國年>_<科目>.md` 一致。
2. **預測題數依該科實際卷型**：混合卷（師消防法規、士各科）申論 2 題；師之純申論卷比照歷屆 4 題，使用者指定單一考點時可只出該考點 1 題。
3. 檔首狀態列固定兩行，之後不再重複。

## 二、每題三區塊（順序固定：題目 → 答題架構 → 擬答）

```markdown
## 預測題一：{考點中文名}

**題目**（預測理由：{一句：頻率／週期／修法依據}）

一、{題幹，仿歷屆情境化敘述}
　　（一）{子題}（{配分} 分）
　　（二）{子題}（{配分} 分）

**答題架構**

1. 破題定義：{要定義的名詞與其出處}
2. 法源依據：{本題核心條文清單}
3. 分項論述：{（一）（二）各自的推導骨架}
4. 結論／實務建議：{收束方向}

**擬答**

（一）{子題標題}

{依「要件 → 法源 → 涵攝 → 小結」逐段推進；每一論點行尾附（法規簡稱 §條號）；
首次引用用全名＋版本日期。計算：公式先標法源 → 代值逐項標來源 → 附單位逐步演算 → 結果檢核。}

（二）…

**本題法源清單**

- 《{法規全名}》（{版本日期} 修正）§{條號}：{一句該條在本題的作用}
```

1. **題目區塊比照 `試卷格式.md` 申論題版面**（國字題號、全形括號子題、各標配分）；括號附一句「預測理由」，讓使用者知道為何猜這題。
2. **擬答內文不用條列堆砌**：以段落推進（要件→法源→涵攝→小結），僅計算演算與法源清單用條列。
3. 無法源內容照批改輸出格式之標注規則：`（非法源：實務見解）`／`（非法源：計算推導）`；修法相關考點，**修法理由寫進擬答首段背景**。
4. **本題法源清單**每題附一次，彙整該題全部引用供使用者逐條查證。

## 三、通用規則

- 公式一律 LaTeX；條文引用保留「條、項、款、目」階層。
- 全篇無 emoji、無表格。
- 本模式產出後即結束，不等使用者作答；文件末尾附一行指路：`想先自己寫再對答案 → 對 Claude 說「用出題模式考我這題」。`
